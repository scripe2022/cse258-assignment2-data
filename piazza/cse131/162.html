We can still  do a left factoring here.

\n

Assume the grammar for an operand is (variables and function names are recognized as ID)

\n
\noperand : ID\n\t\t\t\t| ID ( expression );
\n

After left factoring, it's

\n
\noperand: ID tail;\ntail: \n\t\t| ( expression );
\n

Now we can parse it by looking ahead only one token. And we just change the grammar of operand, which doesn't affect the precedence and associativity for operators.
