Several students raise this question after Tuesday's class. I will  also explain this in class tomorrow. 

\n

1. for <1, x> 1 is the key and x is the value. Similarly for <x, 1> x is the key and 1 is the value.

\n

2. We build 2 hashtables along the way. The first hashtable with value number as its value (e.g., <x, 1> <y, 2>, <1+2,3>) is to record distinct values and thus identify the redundant computations. But after identifying such redundant computation, you still need to remove it. The second hashtable with variable name as its value  (e.g., <3, a>, <1,x>) is for the removal of redundant computation. In this example, after you find that c has a value number of 3 via the use of the first hashtable, you need to refer to the second hashtable to find which variable has such value, and it turns out to be a, and thus you could replace the c=x+z with a.

\n

3. I hope the answers to question 1 and 2 resolve this question.
