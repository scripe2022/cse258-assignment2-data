No! I don't know what the best value of K is for this assignment, but it very well could be 1. The "best" value will be a balance between expressive power (large K) and not overfitting (small K). It costs *a lot* of parameters to have a large value of K, so in a sparse dataset (not many ratings per user / item) the best value of K could indeed be small.
